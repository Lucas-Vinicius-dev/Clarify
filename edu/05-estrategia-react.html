React 19

A use() API no AuthContext

React 19 introduziu a use() API como alternativa ao useContext(). O Clarify já adota isso no AuthContext.

use() em vez de useContext()

// src/context/AuthContext.tsx:165-171 export function useAuth(): AuthContextValue { const ctx = use(AuthContext) if (!ctx) throw new Error( 'useAuth deve ser usado dentro de um AuthProvider' ) return ctx }
Diferença: use() funciona tanto em Server Components quanto Client Components. Além disso, use() pode aceitar promises (para Suspense) além de contextos. O AuthContext usa use() com contexto.

Effects

Mapa de todos os useEffect no projeto

Effects são usados com moderação. A maioria das buscas de dados vai via React Query, não via effect. Aqui estão todos os lugares onde effects aparecem e o que cada um faz:

AuthContext.tsx
[supabase, fetchProfile]
Inicialização da sessão: Chama supabase.auth.getSession() ao montar para restaurar sessão. Depois inscreve-se em onAuthStateChange para reagir a login/logout em tempo real. Cleanup: subscription.unsubscribe().
useDebounce.ts
[value, delay]
Debounce genérico: Toda vez que o valor muda, configura um setTimeout com o delay. Se o valor mudar de novo antes do timeout, o timeout anterior é limpo (clearTimeout).
DrawerMobile.tsx
[drawerOpen]
Travar scroll quando drawer aberto: Adiciona/remove overflow-hidden no body.
FiltroTurmas.tsx
[busca, ...]
Busca com debounce + click-outside: Sincroniza o valor debounced do input. Adiciona listener de click-outside.
LandingPage
[]
Scroll reveal + IntersectionObserver: Animação de scroll-reveal. Cleanup dos observers.
ModalEditarTurma
[turma]
Reset do formulário: Preenche dados da turma no formulário quando o modal abre.
Regra: useEffect não é usado para buscar dados do servidor. Isso é responsabilidade do React Query. Effects são apenas para sincronização com sistemas externos.
Hooks custom

Padrões de hooks customizados

Todos os hooks de dados seguem o mesmo padrão. Hooks de UI são diferentes.

Padrão: Hook de dados

export function useRecurso(opcoes?) { // 1. Query key única const queryKey = ['recurso', opcoes?.filtro] // 2. useQuery com fetch const { data = [], isLoading } = useQuery({ queryKey, queryFn: async () => { const res = await fetch(`/api/recurso?${params}`) return (await res.json()).map(mapRow) }, }) // 3. Mutations com invalidate const criarMutation = useMutation({ mutationFn: async (dados) => { ... }, onSuccess: () => queryClient.invalidateQueries({ queryKey: ['recurso'] }), }) // 4. Retorno return { recursos: data, loading, criar: criarMutation.mutateAsync } }
src/hooks/useDemandas.ts • useTurmas • useUsuarios • useChaves

Padrão: Hook de utilidade

// src/hooks/useDebounce.ts export function useDebounce<T>(value: T, delay: number): T { const [debounced, setDebounced] = useState(value) useEffect(() => { const timer = setTimeout(() => setDebounced(value), delay) return () => clearTimeout(timer) }, [value, delay]) return debounced }

Promises

Como o Clarify lida com assincronicidade

Todo dado é buscado via fetch() + await. Não usamos axios, não usamos biblioteca de fetch adicional.

AuthContext Promise pattern getSession().then() / onAuthStateChange subscription
→
React Query queryFn async/await Toda queryFn retorna Promise resolvida pelo RQ
→
mutateAsync Promise-based mutations Usamos mutateAsync (não mutate) para controle
Não usamos Suspense ou Streaming ainda
Detalhe: O AuthContext usa .then() para manter o fluxo com useCallback. Os hooks de dados usam async/await dentro de queryFn. As mutações retornam mutateAsync (em vez de mutate) para que o componente chamador possa await e tratar erros com try/catch.
