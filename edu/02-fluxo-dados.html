Fluxo de Dados: Componente → Supabase

O caminho completo que um dado percorre: do componente React até o PostgreSQL no Supabase, passando por hooks, API routes e três tipos de cliente Supabase. Tudo é client-side — não há Server Components (exceto o layout root).

Três clientes

Cada contexto, um cliente Supabase diferente

O Supabase tem três "sabores" de cliente, cada um com permissões e use case diferentes:

Browser

createBrowserClient()

src/lib/supabase/client.ts

Usado em componentes client-side que acessam o Supabase diretamente, sem passar por API route. Exemplo: AuthContext.tsx chama supabase.auth.getSession() e supabase.auth.onAuthStateChange() no navegador.

RLS (Row Level Security) ativo — o cliente só vê dados que o usuário logado tem permissão.
Server

createServerClient()

src/lib/supabase/server.ts

Usado nas API routes (/api/*). Lê os cookies da requisição (next/headers) para obter a sessão do usuário e fazer queries autenticadas. A maioria das chamadas de dados passa por aqui.

Também respeita RLS — usa a sessão extraída dos cookies.
Admin

createAdminClient()

src/lib/supabase/admin.ts

Usa a service-role key do Supabase. Bypassa RLS completamente. Usado em operações privilegiadas: criar usuário no registro (supabase.auth.admin.createUser), gerenciar chaves de ativação, criar/deletar perfis, upload/remoção de anexos (Storage + signed URLs).

Cuidado: Esse cliente tem acesso total ao banco. Usado em /api/auth/register, /api/chaves, /api/perfis (POST/DELETE), /api/demandas (POST), /api/demandas/:protocolo/anexos.

Escrita

Como funciona uma mutação (com optimistic update)

O caso mais completo é a atualização de status de demanda. O Clarify usa optimistic update para que a UI responda instantaneamente, e só mostra erro se o servidor realmente falhar.

1. Clique Aprovar demanda ModalFeedback.tsx
→
2. Hook demandas.atualizarStatus() useDemandas.ts
→
3. API PATCH /api/demandas/:protocolo route.ts
→
4. Supabase UPDATE demandas SET status
onMutate Atualiza cache imediatamente
onError Restaura cache anterior (rollback)
onSettled Revalida query ['demandas'] no servidor 
// src/hooks/useDemandas.ts — Optimistic update pattern

onMutate: async ({ protocolo, novoStatus, feedback }) => {
  // 1. Cancela queries em andamento
  await queryClient.cancelQueries({ queryKey: ['demandas'] })

  // 2. Salva estado anterior para rollback
  const previousQueries = queryClient.getQueriesData({ queryKey: ['demandas'] })

  // 3. Atualiza cache imediatamente
  queryClient.setQueriesData(['demandas'], (old) =>
    old.map(d => d.protocolo === protocolo
      ? { ...d, status: novoStatus, feedback: feedback ?? d.feedback }
      : d
    )
  )

  return { previousQueries } // passado para onError
},

onError: (_err, _vars, context) => {
  // 4. Rollback: restaura cache anterior
  context?.previousQueries?.forEach(([key, data]) =>
    queryClient.setQueryData(key, data)
  )
},

onSettled: () => {
  // 5. Sempre revalida com o servidor no final
  queryClient.invalidateQueries({ queryKey: ['demandas'] })
}

Hooks

Padrão de todos os hooks de dados

Cada hook de recurso segue o mesmo padrão. Se você entende um, entende todos.

1. useQuery queryKey + queryFn useQuery({ queryKey, queryFn }) → { data, isLoading }
→
2. useMutation criar / atualizar / excluir useMutation({ mutationFn, onSuccess: invalidateQueries })
→
3. Retorno Objeto com dados + ações { data, loading, criar, atualizar, recarregar, filtrar }
Exemplo: useDemandas, useTurmas, useUsuarios, useChaves, usePerfil 
// Padrão de todos os hooks (src/hooks/use*.ts)

export function useRecurso(opcoes?) {
  const queryClient = useQueryClient()
  const queryKey = ['recurso', opcoes?.filtro]  // único queryKey

  // Leitura com cache
  const { data = [], isLoading } = useQuery({
    queryKey,
    queryFn: async () => {
      const res = await fetch(`/api/recurso?${params}`)
      return (await res.json()).map(mapRow)
    },
  })

  // Mutação com invalidação automática
  const criarMutation = useMutation({
    mutationFn: async (dados) => {
      const res = await fetch('/api/recurso', { method: 'POST', body: JSON.stringify(dados) })
      return await res.json()
    },
    onSuccess: () => queryClient.invalidateQueries({ queryKey: ['recurso'] }),
  })

  return {
    recursos: data,
    loading,
    criar: criarMutation.mutateAsync,
    recarregar: () => queryClient.invalidateQueries({ queryKey: ['recurso'] }),
  }
}
